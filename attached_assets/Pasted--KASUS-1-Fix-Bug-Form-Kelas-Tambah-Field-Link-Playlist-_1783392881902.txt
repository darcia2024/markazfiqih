## BUG FIX

Ada function yang mengubah data kelas (objek `cls`) menjadi form state, dipanggil
saat tombol "Edit" pada sebuah kelas diklik. Saat ini function tersebut
hardcode dua field berikut jadi string kosong, alih-alih mengambil dari data
kelas yang sebenarnya:

  gdriveMateriUrl: '',
  waGroupUrl: '',

Perbaiki agar mengambil dari data asli:

  gdriveMateriUrl: cls.gdriveMateriUrl ?? '',
  waGroupUrl: cls.waGroupUrl ?? '',

Bug ini menyebabkan: setiap kali admin membuka kelas manapun untuk diedit
(walau cuma mau ubah judul misalnya) lalu submit, link Google Drive materi
dan link Grup WhatsApp kelas tersebut ikut terhapus karena dikirim ulang
sebagai string kosong ke backend.

## FITUR BARU — Field Link Playlist YouTube

Tambahkan input baru "Link Playlist YouTube" di form tambah/edit kelas.
Field ini WAJIB ada karena kolom youtubePlaylistId sudah ada di database
dan backend (lihat lib/db/src/schema/classes.ts dan
artifacts/api-server/src/routes/classes.ts) tapi form admin belum punya
input untuk mengisinya sama sekali — jadi sekarang admin tidak bisa
menempelkan link YouTube ke kelas manapun.

Detail implementasi:

1. Tambahkan ke type ClassFormState:
   youtubePlaylistId: string;

2. Tambahkan ke EMPTY_FORM:
   youtubePlaylistId: '',

3. Di function yang convert data kelas ke form state (yang dibenerin di
   BUG FIX di atas), tambahkan juga:
   youtubePlaylistId: cls.youtubePlaylistId ?? '',

4. Tambahkan elemen form baru, letakkan TEPAT SEBELUM blok
   "Link Google Drive Materi (PDF)" yang sudah ada, dengan struktur JSX
   dan styling yang identik (Label + Input), seperti ini:

   <div className="space-y-2">
     <Label htmlFor="class-youtube-playlist">Link Playlist YouTube</Label>
     <Input
       id="class-youtube-playlist"
       type="url"
       placeholder="https://www.youtube.com/playlist?list=..."
       value={form.youtubePlaylistId}
       onChange={(e) => setForm((p) => ({ ...p, youtubePlaylistId: e.target.value }))}
       data-testid="input-class-youtube-playlist"
     />
   </div>

5. Di handler submit, tambahkan ke payload yang dikirim ke backend:
