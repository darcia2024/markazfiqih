# Prompt 22: Fitur "Paket Bundle" — Halaman Terpisah + Cart & Checkout Terintegrasi

## Konteks
Bundle rekaman adalah paket beberapa kelas yang dijual dengan 1 harga
diskon gabungan. Data dari klien (dokumen `Harga_Kelas.docx`):

| Bundle | Kelas di dalamnya | Harga Normal | Harga Bundle |
|---|---|---|---|
| Fiqih Thaharah | Fiqih Najis, Fiqih Wudhu Mandi Tayammum | 110.000 | 90.000 |
| Fiqih Shalat | Fiqih Shalat Jama'ah, Fiqih Shalat & Khutbah Jum'at, Fiqih Safar | 170.000 | 140.000 |
| Fiqih Ibadah | Fiqih Shalat Jama'ah, Fiqih Shalat & Khutbah Jum'at, Fiqih Safar, Fiqih Puasa, Fiqih Zakat Fitri, Fiqih Haji Umrah | 385.000 | 310.000 |
| Fiqih Pernikahan | Fiqih Nikah, Fiqih Talak, Fiqih Maulud | 190.000 | 150.000 |
| Seluruh Rekaman | Semua 16 kelas berbayar (lihat catatan di bawah) | 1.090.000 | 800.000 |

Bundle bisa langsung ditambahkan ke keranjang sebagai 1 item, dan saat
checkout, user ter-enroll ke **semua kelas** di dalam bundle itu sekaligus.

Bundle ditampilkan di **halaman terpisah** dari katalog kelas biasa (bukan
bercampur dengan kelas individual).

Jalankan section per section, jangan digabung.

## Section 1 — Skema Database (SQL manual, jalankan di Supabase SQL Editor)

### 1a. Tabel bundle
```sql
CREATE TABLE bundles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  description TEXT NOT NULL DEFAULT '',
  normal_price INTEGER NOT NULL,
  bundle_price INTEGER NOT NULL,
  status TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft', 'published')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE bundle_classes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  bundle_id UUID NOT NULL REFERENCES bundles(id) ON DELETE CASCADE,
  class_id UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  UNIQUE (bundle_id, class_id)
);

ALTER TABLE bundles ENABLE ROW LEVEL SECURITY;
ALTER TABLE bundle_classes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public can view published bundles"
  ON bundles FOR SELECT
  USING (status = 'published');

CREATE POLICY "Public can view bundle classes"
  ON bundle_classes FOR SELECT
  USING (true);
```

### 1b. Ubah `cart_items` supaya bisa menyimpan bundle
```sql
ALTER TABLE cart_items ADD COLUMN bundle_id UUID REFERENCES bundles(id) ON DELETE CASCADE;
ALTER TABLE cart_items ALTER COLUMN class_id DROP NOT NULL;
ALTER TABLE cart_items ADD CONSTRAINT cart_items_class_or_bundle_check
  CHECK (
    (class_id IS NOT NULL AND bundle_id IS NULL) OR
    (class_id IS NULL AND bundle_id IS NOT NULL)
  );

-- Ganti unique constraint lama supaya juga cover kombinasi user+bundle
ALTER TABLE cart_items DROP CONSTRAINT IF EXISTS cart_items_user_class_unique;
CREATE UNIQUE INDEX cart_items_user_class_unique ON cart_items (user_id, class_id) WHERE class_id IS NOT NULL;
CREATE UNIQUE INDEX cart_items_user_bundle_unique ON cart_items (user_id, bundle_id) WHERE bundle_id IS NOT NULL;
```

### 1c. Ubah `invoice_items` supaya bisa mencatat bundle yang dibeli
```sql
ALTER TABLE invoice_items ADD COLUMN bundle_id UUID REFERENCES bundles(id) ON DELETE SET NULL;
ALTER TABLE invoice_items ALTER COLUMN class_id DROP NOT NULL;
```
(class_id tetap diisi per-kelas di dalam bundle saat invoice dibuat — lihat
Section 3 — supaya histori pembelian tetap granular per kelas, kolom
`bundle_id` di sini hanya sebagai penanda "kelas ini didapat lewat bundle
apa", bukan pengganti class_id.)

## Section 4 — Edge Function `checkout` Menangani Bundle

Edit `supabase/functions/checkout/index.ts`:

- Saat mengambil cart items, join juga ke `bundles` dan `bundle_classes`
  untuk item yang punya `bundle_id`.
- Untuk **item kelas biasa**: perilaku tetap seperti sekarang (1
  `invoice_items` row per kelas, harga dari `discount_price` atau
  `base_price`).
- Untuk **item bundle**: harga yang dipakai untuk total adalah
  `bundle_price` dari tabel `bundles` (bukan menjumlahkan harga individual
  tiap kelas). Tapi tetap buat **satu `invoice_items` row per kelas di
  dalam bundle** (bukan 1 row untuk bundle-nya), dengan:
  - `class_id` = id kelas tersebut
  - `bundle_id` = id bundle asalnya
  - `price` = bagi rata `bundle_price` ke semua kelas dalam bundle itu
    (misal bundle 90.000 untuk 2 kelas → masing-masing dicatat 45.000),
    supaya total invoice tetap benar dan histori tetap granular per kelas.
    Kalau pembagian tidak bulat, sisa pembulatan boleh ditambahkan ke
    kelas pertama dalam bundle supaya total tetap pas.

Update juga edge function `simulate-success` (`supabase/functions/
simulate-success/index.ts`) — enrollment sudah otomatis benar karena tetap
berbasis `class_id` per row di `invoice_items` (tidak perlu logic khusus
tambahan untuk bundle di sini, karena Section ini sudah memastikan setiap
kelas dalam bundle punya row `invoice_items` sendiri yang akan di-enroll
satu-satu seperti biasa).
