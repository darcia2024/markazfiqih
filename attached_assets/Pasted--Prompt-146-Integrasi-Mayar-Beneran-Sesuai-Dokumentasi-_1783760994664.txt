# Prompt 146: Integrasi Mayar Beneran (Sesuai Dokumentasi Terkini)

## Konteks — Temuan Penting Sebelum Mulai
Kode yang sudah ada (`checkout/index.ts`, `mayar-webhook/index.ts`)
disiapkan berdasarkan ASUMSI API Mayar versi lama/berbeda. Setelah dicek
ke dokumentasi resmi Mayar terbaru (docs.mayar.id), ada 3 perbedaan
penting yang HARUS diperbaiki:

1. **Endpoint create invoice yang benar**: `https://api.mayar.id/hl/v1/invoice/create`
   (BUKAN `/payment/create` seperti di kode lama).
2. **Field body request beda total** — Mayar sekarang butuh: `name`,
   `email`, `mobile` (WAJIB, nomor HP — sistem kita SAAT INI TIDAK
   PERNAH mengumpulkan nomor HP user sama sekali, perlu ditambahkan),
   `redirectUrl`, `description`, `expiredAt` (ISO 8601), dan `items`
   (array of `{quantity, rate, description}` — BUKAN `amount` langsung).
3. **Webhook Mayar TIDAK mendukung signature verification (HMAC)** —
   dokumentasi resmi webhook Mayar (docs.mayar.id/integration/webhook)
   TIDAK menyebutkan mekanisme signature/HMAC sama sekali, cuma:
   set URL webhook di dashboard, method POST, body JSON. Kode lama yang
   mengasumsikan header `x-mayar-signature` HARUS diganti — pakai
   pendekatan **token rahasia di URL webhook** sebagai gantinya (lebih
   aman daripada tidak ada verifikasi sama sekali).

const mayarApiKey = Deno.env.get('MAYAR_API_KEY');

## Bagian D — Perbaiki `supabase/functions/mayar-webhook/index.ts`

GANTI TOTAL mekanisme verifikasi dari signature-header jadi
**token rahasia di URL webhook** (karena Mayar tidak mendukung
signature verification):

```ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

Deno.serve(async (req) => {
  if (req.method === 'OPTIONS') {
    return new Response('ok', {
      headers: { 'Access-Control-Allow-Origin': '*', 'Access-Control-Allow-Headers': 'content-type' },
    });
  }

  try {
    // Verifikasi token rahasia dari query parameter URL
    // (URL webhook di dashboard Mayar akan diset dengan token ini, contoh:
    // https://xxx.supabase.co/functions/v1/mayar-webhook?token=RAHASIA_PANJANG_ACAK)
    const url = new URL(req.url);
    const token = url.searchParams.get('token');
    const expectedToken = Deno.env.get('MAYAR_WEBHOOK_TOKEN');

    if (!expectedToken || expectedToken === 'PLACEHOLDER_ISI_NANTI' || token !== expectedToken) {
      return new Response(JSON.stringify({ error: 'Unauthorized' }), {
        status: 401,
        headers: { 'Content-Type': 'application/json' },
      });
    }

    const payload = await req.json();

    // Sesuai dokumentasi resmi Mayar: field event & data.status (boolean)
    if (payload.event !== 'payment.received' || payload.data?.status !== true) {
      return new Response(JSON.stringify({ received: true, skipped: true }), {
        headers: { 'Content-Type': 'application/json' },
      });
    }

    const mayarInvoiceId = payload.data.id;
    const extraData = payload.data.extraData ?? {};
    const invoiceId = extraData.idProd; // ini id invoice internal kita, dikirim balik oleh Mayar

    if (!invoiceId) {
      console.error('Webhook Mayar tidak punya idProd/extraData:', payload);
      return new Response(JSON.stringify({ error: 'Missing invoice reference' }), {
        status: 400,
        headers: { 'Content-Type': 'application/json' },
      });
    }

    const supabaseAdmin = createClient(
      Deno.env.get('SUPABASE_URL') ?? '',
      Deno.env.get('SUPABASE_SERVICE_ROLE_KEY') ?? '',
    );

    const { data: invoice, error: invoiceError } = await supabaseAdmin
      .from('invoices')
      .select('id, user_id, status, invoice_items(class_id, bundle_id, ebook_id)')
      .eq('id', invoiceId)
      .single();
    if (invoiceError || !invoice) throw invoiceError ?? new Error('Invoice tidak ditemukan');

    if (invoice.status === 'paid') {
      // Sudah diproses sebelumnya (webhook Mayar bisa terkirim berkali-kali), aman diabaikan
      return new Response(JSON.stringify({ received: true, already_paid: true }), {
        headers: { 'Content-Type': 'application/json' },
      });
    }

    await supabaseAdmin
      .from('invoices')
      .update({ status: 'paid', mayar_invoice_id: mayarInvoiceId, paid_at: new Date().toISOString() })
      .eq('id', invoiceId);

    // Buat enrollments untuk tiap kelas (ikuti pola yang SUDAH ADA
    // sebelumnya di file ini untuk bagian ini — TIDAK diubah logicnya,
    // cuma dipindah ke sini karena struktur function-nya berubah total)
    const classItems = (invoice.invoice_items as any[]).filter((i) => i.class_id);
    if (classItems.length > 0) {
      const enrollments = classItems.map((item: any) => ({ user_id: invoice.user_id, class_id: item.class_id }));
      await supabaseAdmin.from('enrollments').upsert(enrollments, { onConflict: 'user_id,class_id', ignoreDuplicates: true });
    }

    // Buat ebook_purchases untuk tiap ebook (dari Prompt 119, JANGAN dihapus)
    const ebookItems = (invoice.invoice_items as any[]).filter((i) => i.ebook_id);
    if (ebookItems.length > 0) {
      const ebookPurchases = ebookItems.map((item: any) => ({ user_id: invoice.user_id, ebook_id: item.ebook_id }));
      await supabaseAdmin.from('ebook_purchases').upsert(ebookPurchases, { onConflict: 'user_id,ebook_id', ignoreDuplicates: true });
    }

    return new Response(JSON.stringify({ received: true }), {
      headers: { 'Content-Type': 'application/json' },
    });
  } catch (error) {
    console.error('mayar-webhook error:', error);
    return new Response(JSON.stringify({ error: (error as Error).message }), {
      status: 500,
      headers: { 'Content-Type': 'application/json' },
    });
  }
});
```

**PENTING**: sebelum menimpa logic pembuatan `enrollments`/`ebook_purchases`
di atas, BUKA DULU isi `mayar-webhook/index.ts` yang SEKARANG ADA dan
PASTIKAN semua logic pembuatan enrollment/ebook_purchase yang sudah
dibangun sebelumnya (termasuk dari Prompt 106 soal cegah duplikat, kalau
ada bagian terkait) tetap terbawa/tidak hilang — function di atas cuma
CONTOH STRUKTUR, sesuaikan dengan detail yang sudah teruji sebelumnya.
