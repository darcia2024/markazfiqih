BAGIAN 1 — FIX BUG KRITIS: Kelas tidak masuk ke akun user setelah bayar
===========================================
Ini bug paling penting, kerjakan lebih dulu. Saat ini setelah user membayar 
lewat Mayar, sistem TIDAK memberikan akses kelas ke user tersebut.

Root cause kemungkinan: tidak ada mekanisme yang mengonfirmasi status 
pembayaran balik ke aplikasi setelah user membayar di halaman Mayar.

Solusi yang harus dibuat:
1. Buat endpoint webhook BARU, misal `/api/webhooks/mayar`, yang akan menerima 
   notifikasi dari Mayar. JANGAN langsung percaya isi payload webhook untuk 
   menentukan status pembayaran (karena format signature verification Mayar 
   belum saya pastikan). Sebagai gantinya:
   a. Ambil invoice ID / reference dari payload webhook yang masuk
   b. Panggil balik API Mayar "Get Detail Invoice" (cek endpoint yang sesuai 
      di dokumentasi Mayar untuk get invoice by ID) menggunakan MAYAR_API_KEY 
      untuk mengonfirmasi status invoice tersebut SECARA LANGSUNG dari Mayar 
      (bukan dari payload webhook)
   c. Jika status dari API benar-benar "paid"/"success", baru proses pemberian 
      akses kelas ke user
   d. Log seluruh payload webhook yang masuk (header + body) untuk keperluan 
      debug, karena formatnya belum saya konfirmasi 100%
   e. Selalu return 200 OK ke Mayar setelah selesai memproses, agar tidak 
      di-retry terus menerus

2. Cari di kode yang sudah ada: bagaimana cara sistem menyimpan relasi antara 
   "user membeli kelas apa" (kemungkinan ada tabel orders/enrollments/purchases). 
   Setelah invoice dikonfirmasi "paid", tambahkan/update record supaya user 
   tersebut resmi memiliki akses ke kelas yang dibeli (cek kelas mana yang 
   dibeli dari deskripsi/item invoice atau dari metadata yang dikirim saat 
   create invoice — pastikan saat create invoice, sertakan referensi unik 
   seperti orderId/userId+classId di description atau field lain yang bisa 
   dibaca kembali saat konfirmasi ini)

3. Setelah kode ini jadi, tunjukkan ke saya URL webhook yang harus saya 
   register ke dashboard Mayar (via endpoint Register URL Hook atau dashboard), 
   dan cara saya test end-to-end-nya (dari checkout sampai kelas benar-benar 
   muncul di akun user)

4. PENTING: pastikan proses ini idempotent — kalau webhook yang sama terkirim 
   dua kali (retry dari Mayar), jangan sampai user dapat akses ganda/error

BAGIAN 4 — Fix bug mobile: tombol keranjang menghalangi klik
===========================================
Di tampilan mobile, ada floating bar "1 item di keranjang - Rp xxx" yang 
posisinya fixed di bawah layar. Bar ini MENUTUPI/menghalangi tombol metode 
pembayaran (QRIS, Virtual Account, dst) sehingga user tidak bisa mengklik 
tombol tersebut di mobile, meskipun di desktop/PC tidak masalah karena 
posisinya tidak bertabrakan.

Fix yang diperlukan:
- Cek CSS/styling floating bar tersebut di breakpoint mobile — kemungkinan 
  perlu menambahkan padding-bottom pada container utama halaman checkout 
  sebesar tinggi floating bar tersebut, supaya konten (termasuk tombol 
  metode pembayaran) tidak tertutup
- Pastikan z-index floating bar tetap di atas konten (agar tetap terlihat), 
  TAPI konten di baliknya tidak boleh ter-block dari sisi clickable area
- Test dengan cara scroll ke section "Metode pembayaran" di viewport mobile 
  (gunakan device toolbar/responsive mode) dan pastikan semua tombol metode 
  pembayaran bisa diklik tanpa terhalang
- Jangan mengubah floating bar ini di tampilan desktop jika di sana sudah 
  tidak ada masalah
