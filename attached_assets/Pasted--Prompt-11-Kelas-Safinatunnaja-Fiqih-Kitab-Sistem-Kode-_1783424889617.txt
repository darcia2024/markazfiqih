# Prompt 11: Kelas Safinatunnaja (Fiqih Kitab) + Sistem Kode Voucher

## Konteks
Kelas baru "Safinatunnaja" sudah ready untuk kategori Fiqih Kitab. Harga normal
Rp180.000, harga presale Rp150.000 kalau user masukin kode voucher saat checkout.
Link rekaman (YouTube/Gdrive) belum ada — akan diisi admin belakangan lewat
Admin Panel setelah kelas ini dibuat.

Voucher untuk sekarang **khusus dipakai di kelas ini saja** (bukan sistem
voucher umum lintas kelas) — tapi struktur tabelnya dibuat cukup generic biar
gampang di-reuse nanti kalau ada kelas lain yang butuh voucher serupa.

Jalankan section per section, jangan digabung.

### 1b. Tabel voucher (SQL, jalankan manual juga)
```sql
CREATE TABLE class_vouchers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  code TEXT NOT NULL,
  discount_price INTEGER NOT NULL, -- harga final setelah voucher dipakai
  is_active BOOLEAN NOT NULL DEFAULT true,
  max_uses INTEGER, -- NULL = unlimited
  used_count INTEGER NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (class_id, code)
);

ALTER TABLE class_vouchers ENABLE ROW LEVEL SECURITY;

-- Publik boleh baca voucher aktif untuk validasi kode di frontend
CREATE POLICY "Public can validate active vouchers"
  ON class_vouchers FOR SELECT
  USING (is_active = true);

-- Hanya service role (edge function) yang boleh update used_count
-- (tidak perlu policy tambahan, edge function pakai service role key yang bypass RLS)
```

Contoh insert voucher untuk Safinatunnaja (jalankan manual setelah section 1a,
pakai `class_id` dari kelas yang baru diinsert):
```sql
INSERT INTO class_vouchers (class_id, code, discount_price, max_uses)
VALUES ('GANTI_DENGAN_CLASS_ID_SAFINATUNNAJA', 'PRESALE150', 150000, NULL);
```

## Section 2 — Fungsi voucher di `src/lib/db.ts`

Tambahkan dua fungsi baru di `artifacts/markaz-fiqh/src/lib/db.ts`, ikuti pola
fungsi lain yang sudah ada di file ini (langsung query Supabase, bukan lewat
Express):

1. `validateVoucher(classId: string, code: string)` — query tabel
   `class_vouchers` untuk `class_id` + `code` (case-insensitive, uppercase
   sebelum query) yang `is_active = true`. Kalau `max_uses` tidak null dan
   `used_count >= max_uses`, anggap tidak valid. Return
   `{ valid: true, discountPrice: number } | { valid: false, message: string }`.
   Pesan error untuk kode salah: "Kode voucher tidak valid atau sudah tidak
   berlaku."

2. Cek fungsi `createCheckout` yang sudah ada (baris ~347) — jangan diubah di
   step ini, akan diupdate di Section 3 untuk terima parameter voucher.

## Section 3 — Update Edge Function `checkout` untuk terima kode voucher

Edit `supabase/functions/checkout/index.ts`:

- Terima body request `{ voucherCode?: string }` (parse dari `req.json()`,
  optional).
- Setelah ambil `cartItems`, kalau ada `voucherCode`:
  - Untuk tiap item di cart, cek apakah ada row di `class_vouchers` yang cocok
    `class_id` + `code` (uppercase match) + `is_active = true`.
  - Kalau ketemu dan valid (cek `max_uses` vs `used_count` sama seperti di
    Section 2), pakai `discount_price` dari voucher (override
    `discount_price`/`base_price` kelas itu) untuk hitung `totalAmount` dan
    `invoice_items`.
  - Kalau `voucherCode` diisi tapi tidak match ke kelas manapun di cart,
    return error 400: `{ error: 'Kode voucher tidak valid untuk kelas di keranjang ini.' }`
  - Kalau valid dan dipakai, increment `used_count` pada row voucher tersebut
    (pakai `supabaseAdmin`, bypass RLS).
- Field response tambahan: sertakan `voucherApplied: boolean` di response JSON
  akhir.
- Jangan ubah bagian Mayar integration yang sudah di-comment — biarkan seperti
  itu.

Edit `artifacts/markaz-fiqh/src/pages/CartPage.tsx`:

  (`'idle' | 'valid' | 'invalid'`) di komponen `CartContent`.

  tambahkan input teks kecil + tombol "Pakai" untuk kode voucher, dengan style

  voucher). Kalau tidak valid: tampilkan pesan error merah kecil di bawah
  input, jangan pakai `alert()`.
- Update `handleCheckout` supaya kirim `voucherCode` (kalau ada dan valid) ke
  `createCheckout`.
- Update fungsi `createCheckout` di `src/lib/db.ts` supaya terima parameter
  optional `voucherCode?: string` dan diteruskan sebagai body ke fetch edge

Jangan ubah struktur/layout lain di CartPage selain penambahan blok voucher
