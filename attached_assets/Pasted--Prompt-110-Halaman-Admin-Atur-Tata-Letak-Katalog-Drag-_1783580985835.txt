# Prompt 110: Halaman Admin "Atur Tata Letak Katalog" (Drag-and-Drop)

## Konteks
Urutan kelas per kategori sudah bisa diatur lewat field "Urutan Tampil"
(`display_order`) di form edit kelas (Prompt 89), tapi caranya buka
dialog edit satu-satu dan ketik angka manual â€” kurang praktis kalau mau
susun ulang banyak kelas sekaligus. Buat halaman admin baru khusus:
drag-and-drop kelas langsung untuk atur urutan tampil di Katalog.

Field `display_order` yang dipakai TETAP SAMA seperti yang sudah ada
(TIDAK ada kolom baru) â€” halaman ini cuma cara baru yang lebih praktis
untuk mengubah nilai kolom yang sudah ada itu.

## Bagian B â€” Function di `lib/db.ts`

```ts
export async function bulkUpdateDisplayOrder(
  updates: { id: string; displayOrder: number }[],
): Promise<void> {
  // Jalankan semua update secara paralel â€” cepat untuk jumlah kelas
  // yang wajar (puluhan), tidak perlu batching kompleks
  const results = await Promise.all(
    updates.map((u) =>
      supabase.from('classes').update({ display_order: u.displayOrder }).eq('id', u.id),
    ),
  );
  const firstError = results.find((r) => r.error)?.error;
  if (firstError) throw firstError;
}
```

## Bagian C â€” Halaman `AdminCatalogLayoutPage.tsx`

   kategori yang sama (tidak bisa drag kelas dari satu kategori ke
   kategori lain di halaman ini â€” kalau mau pindah kategori, tetap
   lewat form edit kelas biasa).

3. **Setup `@dnd-kit`**:
   ```tsx
   import {

   } from '@dnd-kit/core';
   ```

4. **Item yang bisa di-drag**, tampilkan cover thumbnail kecil + judul
   kelas + handle drag (icon `GripVertical` dari lucide-react):
   ```tsx
   function SortableClassItem({ cls }: { cls: ClassRow }) {
     const { attributes, listeners, setNodeRef, transform, transition, isDragging } =
       useSortable({ id: cls.id });

       transform: CSS.Transform.toString(transform),
       transition,

         <span className="flex-1 text-sm font-medium">{cls.title}</span>
         <span className="text-xs text-muted-foreground">{cls.status === 'published' ? 'Published' : 'Draft'}</span>
       </div>
     );
   }
   ```

5. **Handler drop**, langsung SIMPAN OTOMATIS ke database begitu drop
   selesai (bukan tombol "Simpan" terpisah â€” sesuai ekspektasi umum
   tool drag-and-drop, user langsung lihat hasilnya tersimpan):
   ```tsx
   function handleDragEnd(event: DragEndEvent, category: string) {
     const { active, over } = event;
     if (!over || active.id === over.id) return;

     setGroupedClasses((prev) => {
       const group = prev[category];
       const oldIndex = group.findIndex((c) => c.id === active.id);
       const newIndex = group.findIndex((c) => c.id === over.id);
       const reordered = arrayMove(group, oldIndex, newIndex);

       // Simpan display_order baru berdasar posisi (kelipatan 10 supaya
       // ada celah untuk sisipan manual di masa depan lewat form edit)
       const updates = reordered.map((c, idx) => ({ id: c.id, displayOrder: (idx + 1) * 10 }));
       bulkUpdateDisplayOrderMutation.mutate(updates);

       return { ...prev, [category]: reordered };
     });
   }
   ```
   Tambahkan mutation dengan `useMutation` + toast sukses/gagal, dan
   `queryClient.invalidateQueries` untuk query classes terkait supaya
   halaman Katalog yang sebenarnya ikut ter-refresh datanya.

6. Loading state & empty state konsisten dengan halaman admin lain.

   yang barusan diubah ikut berubah sesuai.
