## Skema Tabel yang Sudah Ada (JANGAN diubah strukturnya)
```sql
class_vouchers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  code TEXT NOT NULL,
  discount_price INTEGER NOT NULL,
  is_active BOOLEAN NOT NULL DEFAULT true,
  max_uses INTEGER,           -- NULL = tidak dibatasi
  used_count INTEGER NOT NULL DEFAULT 0
)
```
RLS yang sudah aktif: publik boleh SELECT voucher yang `is_active = true`
(dipakai untuk validasi kode di checkout). Untuk INSERT/UPDATE/DELETE dari
admin panel, tambahkan policy admin baru (lihat bagian SQL di bawah).

## Bagian D — SQL: Policy RLS untuk Admin

Siapkan SQL berikut untuk saya jalankan manual di Supabase SQL Editor
(class_vouchers saat ini cuma punya policy SELECT publik, belum ada
policy untuk admin insert/update/delete):

```sql
CREATE POLICY "class_vouchers: admin bisa insert"
  ON class_vouchers FOR INSERT
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM user_profiles
      WHERE user_profiles.user_id = auth.uid()::text
      AND user_profiles.is_admin = true
    )
  );

CREATE POLICY "class_vouchers: admin bisa update"
  ON class_vouchers FOR UPDATE
  USING (
    EXISTS (
      SELECT 1 FROM user_profiles
      WHERE user_profiles.user_id = auth.uid()::text
      AND user_profiles.is_admin = true
    )
  );

CREATE POLICY "class_vouchers: admin bisa delete"
  ON class_vouchers FOR DELETE
  USING (
    EXISTS (
      SELECT 1 FROM user_profiles
      WHERE user_profiles.user_id = auth.uid()::text
      AND user_profiles.is_admin = true
    )
  );

CREATE POLICY "class_vouchers: admin bisa baca semua"
  ON class_vouchers FOR SELECT
  USING (
    is_active = true
    OR EXISTS (
      SELECT 1 FROM user_profiles
      WHERE user_profiles.user_id = auth.uid()::text
      AND user_profiles.is_admin = true
    )
  );
```

Catatan: policy SELECT publik yang lama (`is_active = true`) mungkin
perlu di-drop dulu kalau nama policy-nya bentrok — cek dulu policy yang
sudah ada dengan:
```sql
SELECT policyname FROM pg_policies WHERE tablename = 'class_vouchers';
```
sebelum CREATE POLICY baru, supaya tidak duplikat/konflik. Kalau ada
policy SELECT lama dengan nama berbeda, drop dulu dengan
`DROP POLICY IF EXISTS "nama_lama" ON class_vouchers;` sebelum bikin
yang baru, supaya admin bisa lihat voucher non-aktif juga (voucher
non-aktif tetap perlu tampil di admin panel biar bisa diaktifkan lagi).
