---
name: Orval-generated Zod schema naming
description: Generated Zod schema/type export names from OpenAPI specs via orval don't always match names implied by the OpenAPI operation/schema naming conventions.
---

When importing Zod schemas/types generated by orval (in `lib/api-zod/src/generated/api.ts`) for use in API routes, do not guess the export name from the OpenAPI spec's naming pattern (e.g. assuming `AddCartItemRequest` or `ListCartItemsParams`).

**Why:** Orval's actual generated names can differ from the intuitive/spec-implied name — e.g. request bodies are suffixed `...Body` not `...Request`, and list query params are suffixed `...QueryParams` not `...Params`. Guessing wrong causes import errors that only surface at build time, costing a debugging cycle.

**How to apply:** Before importing any generated schema/type in a new route or client file, grep `lib/api-zod/src/generated/api.ts` for the actual `export const`/`export type` name first.
